# 🏢 Conference Rooms API

## 📋 Опис проєкту (Бізнес-завдання)
Просте та надійне REST API для управління орендою конференц-залів. Дозволяє додавати зали, послуги, шукати вільні кімнати та створювати бронювання з динамічним розрахунком вартості.

## 💻 Стек технологій
* **.NET 9** (ASP.NET Core Web API)
* **PostgreSQL** (Реляційна база даних)
* **Entity Framework Core (EF Core)** (ORM для роботи з даними)
* **FluentValidation** (Валідація вхідних даних)
* **Swagger** (Документація API)

## 🛠 Технічні рішення (Architecture & Patterns)
* **Clean Architecture на основі сервісів:** Проєкт чітко розділений на шари (Domain, Application, Infrastructure, WebApi) для легкості масштабування, підтримки та тестування.
* **Патерн Repository & Service:** Бізнес-логіка (сервіси) повністю ізольована від логіки доступу до даних (репозиторії).
* **Безпека та цілісність:** * Використання **DTO (Records)**
  * Сувора валідація даних через FluentValidation.
  * Глобальний обробник помилок (**ExceptionHandlingMiddleware**) для безпечної віддачі відповідей клієнту (без розкриття деталей бази даних та stack trace).
* **Продуктивність:** Оптимізовані SQL-запити для аналітичних звітів з використанням агрегації на стороні БД та `.AsNoTracking()`.

## 🧮 Бізнес-правила (Розрахунок вартості)
Вартість оренди розраховується динамічно по годинах:
* `09:00 - 18:00`: Базова вартість.
* `06:00 - 09:00`: Знижка 10%.
* `18:00 - 23:00`: Знижка 20%.
* `12:00 - 14:00`: Націнка 15%.
Уся математика ізольована в окремому хелпері та повністю контролюється на стороні бекенду.

## 🚀 Як запустити проєкт локально
1. Склонуйте репозиторій: `git clone ...`
2. Налаштуйте строку підключення до вашої бази PostgreSQL у `appsettings.json`.
3. Виконайте міграції: `dotnet ef database update`
4. Запустіть проєкт: `dotnet run`
5. Документація API та інтерфейс для тестування доступні за адресою: `https://localhost:<port>/swagger`

## 🚧 Плани на майбутнє 
Через обмежений час на виконання тестового завдання, деякий функціонал не був реалізований. Для повноцінної реалізації проєкту планую додати:
1. **Аутентифікація та авторизація:** Впровадження JWT Policy-based authentication для захисту ендпоінтів та розділення прав (наприклад, Admin та User).
2. **Юніт-тести:** Покриття основної бізнес-логіки (особливо розрахунку вартості оренди та сервісів бронювання) тестами (xUnit та Moq).
3. **Логування:** Налаштування структурованого логування (Serilog) щонайменше в основних контролерах, сервісах.
4. **Пагінація (Pagination):** Додавання пагінації у `GET` запитах для отримання списку залів та звітів, щоб уникнути перевантаження бази даних при великих обсягах інформації.
5. **Кешування:** Використання Memory Cache для кешування частих запитів (наприклад, аналітичних звітів або списку доступних послуг).
